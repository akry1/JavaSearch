In Java, a class definition and its instantiation can be combined into a single step. By doing that the class does not require a name. Those classes are called anonymous classes. An anonymous class can be defined and instantiated in contexts where a reference can be used, and it is a nested class to an existing class. Anonymous class is a special case of a class local to a method; hence they also can access final local variables of the enclosing method.<br />Anonymous classes are most useful to create an instance of an interface or adapter class without needing a brand new class.<br /><br /><br /><br />Code listing 4.12: ActionListener.java<br /><br /><br />1 public interface ActionListener {<br />2     public void actionPerformed();<br />3 }<br /><br /><br /><br /><br /><br /><br /><br />Code section 4.21: Anonymous class.<br /><br /><br />1 ActionListener listener = new ActionListener() {<br />2         public void actionPerformed() {<br />3             // Implementation of the action event<br />4             ...<br />5             return;<br />6         }<br />7     };<br /><br /><br /><br /><br />In the above example the class that implements the ActionListener is anonymous. The class is defined where it is instantiated.<br />The above code is harder to read than if the class is explicitly defined, so why use it? If many implementations are needed for an interface, those classes are used only in one particular place, and it would be hard to come up with names for them, using an anonymous inner class makes sense.<br />The following example uses an anonymous inner class to implement an action listener.<br /><br /><br /><br />Code listing 4.13: MyApp.java<br /><br /><br /> 1 import java.awt.Button;<br /> 2 import java.awt.event.ActionEvent;<br /> 3 import java.awt.event.ActionListener;<br /> 4 <br /> 5 class MyApp {<br /> 6    Button aButton = new Button();<br /> 7 <br /> 8    MyApp() {<br /> 9        aButton.addActionListener(new ActionListener() {<br />10                public void actionPerformed(ActionEvent e) {<br />11                    System.out.println("Hello There");<br />12                }<br />13            }<br />14        );<br />15    }<br />16 }<br /><br /><br /><br /><br />The following example does the same thing, but it names the class that implements the action listener.<br /><br /><br /><br />Code listing 4.14: MyApp.java<br /><br /><br /> 1 import java.awt.Button;<br /> 2 import java.awt.event.ActionEvent;<br /> 3 import java.awt.event.ActionListener;<br /> 4 <br /> 5 class MyApp {<br /> 6    Button aButton = new Button();<br /> 7 <br /> 8    // Nested class to implement the action listener<br /> 9    class MyActionListener implements ActionListener {<br />10        public void actionPerformed(ActionEvent e) {<br />11            System.out.println("Hello There");<br />12        }<br />13    }<br />14    MyApp() {<br />15        aButton.addActionListener(new MyActionListener());<br />16    }<br />17 }<br /><br /><br /><br /><br />Using anonymous classes is especially preferable when you intend to use many different classes that each implement the same interface.<br /><br /><br /><br /> Class Loading<br />Java Programming<br />Nested Classes<br />Generics
