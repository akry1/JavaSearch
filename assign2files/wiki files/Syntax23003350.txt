A documentation comment is framed by slash-star-star and star-slash (i.e. /** ... */). The documentation is in the HTML format.<br /><br /><br /><br />Code listing 8.1: Example.java<br /><br /><br />1 /**<br />2  *  A class to give an <b>example</b> of HTML documentation.<br />3  */<br />4 public class Example {<br />5     /** ...Documentation of a member with the type integer named example... */<br />6     public int example;<br />7 }<br /><br /><br /><br /><br />A documentation comment is placed just above the commented entity (class, constructor, method, field).<br />In a documentation comment, the first part is a description text in the HTML format. The second part is a list of special attributes whose name starts with an at sign (@):<br /><br /><br /><br />Code section 8.1: Documentation comment.<br /><br /><br />1 /**<br />2  *  Get the sum of two integers.<br />3  *  @param a The first integer number.<br />4  *  @param b The second integer number.<br />5  *  @return The value of the sum of the two given integers.<br />6  */<br />7 public int sum(int a, int b) {<br />8     return a + b;<br />9 }<br /><br /><br /><br /><br /><br />Get the sum of two integers.<br />Description of the sum method.<br />@param a The first integer number.<br />Description attribute of the parameter a of the method.<br />@param b The second integer number.<br />Description attribute of the parameter b of the method.<br />@return The value of the sum of the two given integers.<br />Description attribute of the value returned by the method.<br /><br />Here is a non exhaustive list of special attributes:<br /><br /><br />Attribute and syntax<br />In a comment of ...<br />Description<br /><br /><br />@author author<br />class<br />Name of the author of the class.<br /><br /><br />@version version<br />class<br />Version of the class.<br /><br /><br />@deprecated description<br />class, constructor, method, field<br />Flags the entity as deprecated (old version), describes why and by what replace it.<br />If the entity flagged as deprecated by this attribute is used, the compiler give a warning.<br /><br /><br /><br />@see reference<br />class, constructor, method, field<br />Add a link in the section "See also".<br /><br /><br />@param id description<br />constructor and method<br />Describes the method parameter.<br /><br /><br />@return description<br />method<br />Describes the value returned by the method.<br /><br /><br />@exception type description<br />constructor and method<br />Describes the reason of the throw of an exception of the specified type (throws clause).<br /><br /><br />See also annotations since Java 5.
